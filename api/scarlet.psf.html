class scarlet.psf.DeconvolvedObservation(images, psfs=None, weights=None, wcs=None, channels=None, padding=10)[source]¶

Bases: scarlet.observation.Observation

Deconvolved image using a deconvolution kernel

This is the same as ~scarlet.Observation except that the matching algorithm uses a precalculated deconvolution kernel and the images are deconolved using that kernel. A separate class exists to make it less likely that the user executes this operation more than once, which will have adverse affects on the images property.

class scarlet.psf.PSFDiffKernel(frame, initial, band, step=0.01)[source]¶

Bases: scarlet.component.Component

PSF Difference Kernel Source

“Source” used to model the difference kernel to match one PSF to another. Typically this should be modeling the deconvolution kernel to convolve an image from an observed PSF into a model PSF, as the difference from model PSF to observed PSF is calculated much more quickly using a DFT in ~scarlet.Observation.match.

match(self, psfs)[source]¶

Implement the observed PSFs as the difference kernel

This is different than ~scarlet.Observation.match, where the difference kernel that matches the model PSF to the observed PSF is calculated and stored. For a PsfObservation the input PSF, which should be the observed PSF (see below) is stored as the “difference kernel” while the actual deconvolution (difference) kernel is calculated by the model.

Parameters
psfs: `~numpy.array`

The input PSF that is being convolved. For deconvolution this is the observed PSF, since the observed PSF is convolved with the deconvolution kernel to match the model PSF. For matching a model PSF to a wider observed PSF use the Observation class, which calculates the difference kernel much more quickly using a DFT.

Returns
self: ~scarlet.PsfObservation
