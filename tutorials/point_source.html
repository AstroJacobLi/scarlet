Point Source TutorialÂ¶

This is a quick demonstration of how to model both extended objects and point sources in the same scence. After more testing we hope to replace this with a more robust demonstration of crowded field photometry, using an iterative detection/deblending procedure. In the meantime feel free to create your own algorithm for crowded fields and let us know how it goes on the DESC Blending channel on slack.

First we load a simulated image where we know the true value of all of the objects. This allows us to know which sources are galaxies and which ones are stars so we can use the appropriate source type.

Fit the PSFsÂ¶

To avoid artifacts that arise when performing a full deconvolution on exteneded objects, we choose a small âtargetâ PSF and calculate the difference kernel to match the PSF in each band to the target. See Matching PSFâs for a more detailed explanation of PSF matching.

[3]:

# Get the target PSF to partially deconvolve the image psfs
target_psf = scarlet.psf.fit_target_psf(psfs, scarlet.psf.moffat)[0]
target_rgb = scarlet.display.img_to_rgb(target_psf, norm=pnorm)
# Display the target PSF using the same scaling as the observed PSFs
plt.imshow(target_rgb)
plt.show()

Initialize the sourcesÂ¶

Here is where we define the sources. In this case we know which ones are stars and which are galaxies, which may or may not be realistic depending on the depth and locations of the images taken. For example, at depths shallower than GAIA it should be possible to flag most of the stars in regions outside the galactic bulge, while in more crowded fields or long exposures a different method might be needed (such as color priors on stars vs galaxies) to determine which sources to model as point sources and which ones to model as extended objects.

[5]:

# Initalize the sources
sources = []
for idx in np.unique(catalog["index"]):
    src = catalog[catalog["index"]==idx][0]
    if src["is_star"]:
        new_source = scarlet.PointSource(
            (src["y"], src["x"]),
            scene,
            observation,
            fix_morph=True, # This prevents the source morphology from changing
        )
    else:
        new_source = scarlet.ExtendedSource(
            (src["y"], src["x"]),
            scene,
            observation,
            bg_rms=bg_rms,
        )
    sources.append(new_source)

Notice that we fix the morphology (fix_morph=True) and normalize the \(S\) matrix for point sources. This allows us to only set the central pixel and never update it. This is described in more detail in the User Guide. Youâll also notice that we passed the point sources the full PSF in each band, whereas for the extended sources we pass the difference kernel. This is because a point source is a fully deconvolved representation of the object while extended sources are partially deconvolved. It also means that we have to be careful if we have a scene like this that is a mixture of extended sources and point sources. In order to have a consistent sparse (partially deconvolved) representation of the scene we need to convolve the point sources with the target PSF, not the full PSF that matches the image, once deblending has been completed to generate our model.

Also notice that we used the same \(S\) normalization for both the point sources and extended sources, which allows us to have a consistent definition of \(A\) and \(S\) for all sources, regardless of their type.

        plt.plot(src["x"], src["y"], "bx", mew=2)

Our three stars (the red xâs) are initialized to match their peak value with the peak of the image while the extended sources are initialized in the usual way.

The dipoles we see at the center of the stellar sources is because the catalog offset the centers of all the sources by a random amount and by fixing the morphology we also cannot adjust the position, which also caused the code to require a greater number of iterations for convergence. A future version of scarlet will hopefully address this unfortunate side effect of fitting PSF stars.
